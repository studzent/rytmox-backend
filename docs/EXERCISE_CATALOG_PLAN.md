# План каталога упражнений RYTM0X

## Цель

Создать структурированную базу упражнений для:
- Генерации AI-тренировок
- Форм-чека упражнений
- Персонализации программ тренировок

## Минимальный этап (MVP)

### Объём: 30–50 базовых упражнений

Начальный набор должен покрывать основные потребности пользователей и позволить AI генерировать полноценные тренировки.

### Категории по оборудованию

1. **Bodyweight (упражнения с собственным весом)**
   - Отжимания (классические, широкие, узкие)
   - Приседания (классические, на одной ноге)
   - Планки (классическая, боковая)
   - Выпады
   - Бёрпи
   - Скручивания
   - Подъёмы ног
   - И другие базовые движения

2. **Гантели (Dumbbells)**
   - Жим гантелей лёжа/стоя
   - Разводки гантелей
   - Тяга гантелей в наклоне
   - Приседания с гантелями
   - Выпады с гантелями
   - Жим гантелей сидя/стоя
   - Подъёмы гантелей через стороны/вперёд

3. **Штанга (Barbell)**
   - Жим лёжа
   - Становая тяга
   - Приседания со штангой
   - Тяга штанги в наклоне
   - Жим стоя/сидя

4. **Базовый турник**
   - Подтягивания (классические, широким хватом, узким)
   - Вис на турнике
   - Подъёмы ног в висе

### Покрытие мышечных групп

Минимальный набор должен включать упражнения для:

- **Грудь**: отжимания, жим лёжа, разводки
- **Спина**: подтягивания, тяги, становая тяга
- **Ноги**: приседания, выпады, становая тяга
- **Плечи**: жимы, подъёмы, отжимания в стойке
- **Ягодицы**: приседания, выпады, ягодичный мостик
- **Кор (пресс)**: планки, скручивания, подъёмы ног

## Структура таблицы exercises

Таблица `exercises` в базе данных имеет следующую структуру (подробнее см. [DB_SCHEMA.md](./DB_SCHEMA.md)):

- `id` (UUID) — уникальный идентификатор
- `name` (TEXT) — название упражнения
- `description` (TEXT) — описание техники выполнения
- `muscle_groups` (TEXT[]) — массив групп мышц (например: `['chest', 'triceps']`)
- `equipment` (TEXT) — необходимое оборудование (например: `'bodyweight'`, `'dumbbells'`, `'barbell'`, `'pull-up bar'`)
- `created_at` (TIMESTAMP) — дата создания записи

### Примеры значений

**muscle_groups** могут быть:
- `chest`, `back`, `legs`, `shoulders`, `glutes`, `core`, `biceps`, `triceps`, `forearms`, `calves`

**equipment** может быть:
- `bodyweight` — без оборудования
- `dumbbells` — гантели
- `barbell` — штанга
- `pull-up bar` — турник
- `kettlebell` — гиря (в будущем)
- `resistance bands` — резиновые ленты (в будущем)
- `gym` — тренажёрный зал (для сложных тренажёров)

## План расширения

### Этап 1: Расширение до 100+ упражнений

После MVP постепенно добавлять:

- Вариации базовых упражнений (разные углы, хваты, темп)
- Упражнения с дополнительным оборудованием (гири, резиновые ленты)
- Изолирующие упражнения для детальной проработки
- Упражнения для реабилитации и восстановления

### Источники вдохновения

При расширении каталога можно брать идеи из топовых фитнес-приложений:

- **Growl** — структура и категоризация
- **Freeletics** — bodyweight упражнения и вариации
- **Strong** — логика записи подходов и прогресса
- **Fitbod** — персонализация и адаптация

**Важно**: не копировать упражнения 1-в-1, а создавать свои названия и описания, адаптированные под русскоязычную аудиторию и контекст RYTM0X.

### Будущие улучшения структуры

Для более продвинутого каталога можно добавить в таблицу `exercises` дополнительные поля:

- **unilateral/bilateral** — одностороннее/двустороннее упражнение
- **tempo** — рекомендуемый темп выполнения (например: `'2-1-2-0'` для эксцентрик-пауза-концентрик-отдых)
- **difficulty_level** — уровень сложности (beginner, intermediate, advanced)
- **technique_tips** — советы по технике
- **common_errors** — частые ошибки при выполнении
- **video_url** — ссылка на видео-демонстрацию (в будущем)
- **alternative_exercises** — альтернативные упражнения (для замены при отсутствии оборудования)

Эти поля можно добавить как JSONB или отдельные колонки при необходимости.

## Приоритеты заполнения каталога

1. **Высокий приоритет**: базовые многосуставные упражнения (приседания, отжимания, подтягивания, становая тяга)
2. **Средний приоритет**: изолирующие упражнения для каждой группы мышц
3. **Низкий приоритет**: экзотические и узкоспециализированные упражнения

## Интеграция с AI

Каталог упражнений используется AI для:

- Выбора подходящих упражнений на основе целей пользователя
- Учета доступного оборудования
- Балансировки тренировок по группам мышц
- Предложения альтернатив при отсутствии оборудования

Поэтому важно, чтобы каждое упражнение имело:
- Чёткое описание техники
- Корректно указанные группы мышц
- Правильно указанное оборудование
